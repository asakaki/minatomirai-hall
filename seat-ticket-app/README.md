# コンサートチケット 販売状況管理（パイロット版）

座席番号（**階・ブロック・列・席番号**の4要素で一意）ごとの販売記録に基づいて座席表をマークする、静的Webアプリです。Cloudflare Pages にそのまま公開できます（ビルド不要）。

## 機能

1. **座席表表示** — 販売済み座席（グレー）と座席種別（色分け）を反映した一覧表示
2. **販売座席入力** — 階・ブロック・列・席番号を指定して販売登録/取消（販売済みはグレー塗り）
3. **座席種別登録** — SS / S / A / 封鎖席 を座席ごとに登録し、種別ごとに色塗り

座席表は各画面でクリック操作にも対応しています（販売入力＝クリックで販売/取消、種別登録＝選択中の種別をクリックで適用）。

### 凡例（色）

| 種別 | 色 |
|---|---|
| 未設定 | 白 |
| SS席 | ゴールド |
| S席 | ブルー |
| A席 | グリーン |
| 封鎖席 | 斜線（ダークグレー） |
| 販売済 | グレー（種別色を隅に表示） |

## 対象座席（C席・全3階）

PDFの席数に準拠した **1〜3階の C席（合計 1,446席）** を実装。

| 階 | 列 | 席番号 | 席数 |
|---|---|---|---|
| 1階 C | 1〜29列 | 1〜36番 | 1,044席（1-19列=684 / 20-29列=360、境界に区切り線） |
| 2階 C | 1〜6列 | 1〜31番 | 186席 |
| 3階 C | 1〜6列 | 1〜36番 | 216席 |

対象を広げるときは [app.js](app.js) の `SEAT_CONFIG` に階・ブロック・列・席番号を追記するだけで拡張できます（2階のP席や左右バルコニー LA〜LF / RA〜RF なども同様に追加可能）。

## 公演（コンサート）レイヤー

公演ごとに販売状況・座席種別を分けて管理できます。上部の「公演」バーから、
**コンサート名・公演日・開演時間**を登録（追加/編集/削除）し、選択した公演の座席表に切り替わります。

## データ保存（Cloudflare D1）

販売状況・座席種別は **Cloudflare D1（サーバー側のデータベース）** に永続化され、
端末をまたいで共有されます。API（[functions/api/](functions/api/)）経由で読み書きします。

- テーブルは初回リクエスト時に自動作成されます（マイグレーション不要）
- API に接続できない場合（静的ホスティング等）は **自動的に `localStorage` にフォールバック**します
- 座席表表示画面から JSON のエクスポート/インポート、公演単位のリセットが可能です

### スキーマ

| テーブル | 列 |
|---|---|
| `concerts` | id, name, date, time, created_at |
| `seats` | concert_id, seat_key, sold, type（1座席=1行、空になれば自動削除）|

## ローカルで動かす（D1付き）

```bash
# D1 をバインドして Pages Functions ごとローカル起動
npx wrangler pages dev --port 8788 --d1 DB=seat-ticket-db
```

→ http://localhost:8788 。ローカルD1は `.wrangler/state` に作られ、テーブルは自動作成されます。

## Cloudflare Pages へ公開（D1）

```bash
# 1) D1 データベースを作成（発行された database_id を控える）
npx wrangler d1 create seat-ticket-db

# 2) wrangler.toml の [[d1_databases]] の database_id を上記の値に置き換え

# 3) デプロイ（functions/ も一緒に公開される）
npx wrangler pages deploy . --project-name=seat-ticket-pilot
```

ダッシュボードで Git 連携する場合は、Pages プロジェクトの
**Settings → Functions → D1 database bindings** で 変数名 `DB` に作成したD1を割り当ててください
（Build command は空欄、Framework preset は None）。

> 静的のみ（DB不要）で使う場合は `functions/` と `wrangler.toml` を置かなければ、従来どおり `localStorage` 保存で動作します。
